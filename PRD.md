# 极简热量饮食管理小程序 PRD

## 1. 产品概述

**产品名称**: 极简热量饮食管理小程序  
**产品类型**: 微信小程序  
**核心定位**: 帮助用户轻松记录每日饮食热量，提供简洁直观的热量统计和趋势分析

**核心价值主张**:
- 极简操作 - 3秒内完成一次饮食记录
- 清晰直观 - 一眼看清今日热量摄入情况
- 实用统计 - 了解长期热量趋势，科学管理饮食

**目标用户**: 关注健康饮食、需要进行热量控制的人群

## 2. 核心功能

### 2.1 首页（今日概览）

**功能描述**: 展示用户今日的热量摄入总览

**主要元素**:
- 圆形进度环显示今日热量（已摄入 / 目标）
- 剩余可摄入热量数字
- 快速添加按钮（早餐/午餐/晚餐/加餐）
- 今日饮食记录列表（时间+食物+热量）
- 快捷操作：编辑/删除记录

**用户流程**:
1. 用户打开小程序即看到今日概览
2. 点击"添加食物" → 进入食物搜索页
3. 选择食物和份量 → 自动计算热量并保存

### 2.2 食物搜索与记录

**功能描述**: 快速搜索食物并添加记录

**主要元素**:
- 搜索框（支持食物名称模糊搜索）
- 最近使用食物列表
- 常见食物分类（主食/肉蛋/蔬菜/水果/饮品/零食）
- 搜索结果列表（显示食物名称+单位热量）
- 份量选择器（常用份量：100g/份/个/杯等）
- 自定义热量输入

**内置食物数据库**:
- 预置200+常见食物数据
- 每种食物包含：名称、热量(kcal)/100g、常见份量单位
- 用户可自定义添加新食物

### 2.3 统计页面

**功能描述**: 展示热量摄入统计和趋势

**主要元素**:
- 本周热量柱状图（周一至周日）
- 本周平均每日摄入
- 与目标的对比
- 历史记录查询（按日期筛选）
- 月度热量趋势（简单折线图）

### 2.4 个人设置

**功能描述**: 用户基础数据和目标设置

**主要元素**:
- 每日热量目标设置（默认1600kcal，可自定义）
- 个人信息：性别、身高、体重（用于基础代谢计算参考）
- 数据导出（导出CSV格式）
- 清空数据（谨慎操作，需二次确认）
- 关于/使用说明

## 3. 数据模型

### 3.1 用户设置
```javascript
{
  dailyCalorieGoal: Number,      // 每日热量目标，默认1600
  gender: String,                // 'male' | 'female'
  height: Number,                // 身高(cm)
  weight: Number,                // 体重(kg)
  createdAt: String              // 创建时间
}
```

### 3.2 饮食记录
```javascript
{
  id: String,                    // 唯一ID
  date: String,                  // 日期 YYYY-MM-DD
  mealType: String,              // 'breakfast' | 'lunch' | 'dinner' | 'snack'
  foodName: String,              // 食物名称
  calorie: Number,               // 热量(kcal)
  portion: Number,               // 份量
  portionUnit: String,           // 份量单位
  createdAt: String              // 记录时间
}
```

### 3.3 自定义食物
```javascript
{
  id: String,
  name: String,
  caloriePerHundredGram: Number, // 每100g热量
  defaultUnit: String,           // 默认单位
  isCustom: Boolean              // 是否用户自定义
}
```

## 4. UI/UX 设计方向

### 4.1 设计风格
- **风格**: 简约现代、清新自然
- **色调**: 主色为薄荷绿 (#4CAF50系)，辅以暖橙 (#FF9800) 作为热量高低的指示
- **布局**: 底部Tab导航，4个页面（首页/记录/统计/设置）

### 4.2 核心交互原则
- 极简步骤：添加食物最多3步完成
- 即时反馈：操作后立即显示结果
- 滑动操作：列表项支持左滑删除
- 下拉刷新：首页和统计页支持下拉刷新

### 4.3 页面结构
```
├── pages/
│   ├── index/           # 首页 - 今日概览
│   ├── add-food/        # 添加食物/搜索
│   ├── statistics/      # 统计页面
│   └── settings/        # 个人设置
├── components/
│   ├── calorie-ring/    # 热量圆环组件
│   ├── food-item/       # 食物列表项
│   ├── meal-section/    # 餐次分类区块
│   └── date-selector/   # 日期选择器
└── utils/
    ├── data-store.js    # 数据存储
    ├── food-database.js # 食物数据库
    └── format.js         # 格式化工具
```

## 5. 技术方案

### 5.1 技术栈
- **框架**: 原生微信小程序框架
- **样式**: WXSS + CSS变量
- **数据存储**: wx.setStorageSync / wx.getStorageSync
- **图表**: Canvas 手动绘制或使用小程序图表组件

### 5.2 架构设计
- 采用模块化设计，工具函数与页面逻辑分离
- 数据层统一使用 Storage API
- 页面间通过事件总线或 URL 参数传递数据

### 5.3 数据持久化
- 所有数据存储在微信本地存储
- 每日数据按日期组织
- 支持数据导入导出
